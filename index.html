Windows 的 Zip 文件在 MacOS 下文件名乱码问题
开不了学了！！！嘛……突如其来的疫情，现在估计大家都开不了学了，学校已经准备开始上网课了，在接收学校发来的网课文件的时候发现了个严重的问题……解压之后文件名全是乱码！！！记得以前使用 Linux 做主力系统的时候也遇到过这种问题……yami 现在的主力系统是 MacOS ，是基于 Unix 的，系统编码一般默认 UTF-8，和 Linux 一样，然而大多数人用的系统是 Windows ，在中文的 Windows 中默认系统编码是 GB3212，出现这种问题的文件都是 zip 文件…后来从网上查资料知道了原来 zip 没带指定编码格式的头就会是这样……总不能告诉老师用别的工具加个头吧？看起来还是自己解决好了。 怎么解决呢？网上给出的方案大概就几种： 安装可以指定解压编码的解压软件 使用其他压缩格式，rar，7z 等都带了编码头 使用脚本转换 第一种……应该是最简单的方案了吧，但感觉自己也不习惯装这些第三方软件，第二种对自己对情况不太现实，那就只能第三种自己转换啦～ 首先尝试用 Python 获取文件名，然后用 UTF8 编码文件名成原始的二进制数据，使用 GBK 解码数据，输出文
